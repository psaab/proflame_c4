🔥 Proflame WiFi Fireplace Driver

Version: 2026051726
Date: May 17, 2026
Author: Paul Saab
Device: SIT Group Proflame 2 WiFi Controller

Overview

This Control4 driver provides complete control over SIT Group Proflame 2 WiFi enabled fireplaces. It connects directly to the fireplace controller via WebSocket and provides real-time status updates and control.

Key Features:
  • Full power and mode control (Off, Standby, Manual, Smart, Eco)
  • Flame level control (0-6)
  • Fan/blower control (0-6)
  • Accent light control (0-6)
  • Thermostat temperature control (60-90°F)
  • Timer functionality
  • Auxiliary output and split flame control
  • Real-time status monitoring

Installation

  1. Add the driver to your Control4 project via Composer Pro
  2. Configure the IP address of your Proflame WiFi controller
  3. The driver will automatically connect and begin receiving status updates

Configuration Properties

Connection Settings

Property Description Default
IP Address IP address of the Proflame WiFi controller (empty)
Port TCP port number (usually 88) 88
Ping Interval Keep-alive interval in seconds 5
Reconnect Delay Seconds to wait before reconnecting after disconnect 10
Strict WebSocket Handshake Require full WebSocket upgrade header validation instead of compatibility fallback Off

Default Behavior Settings

Property Description Default
Default On Mode Mode to use when turning on: Manual, Smart (Thermostat), or Eco Smart (Thermostat)
Default Flame Level Flame level to set when turning on (1-6) 6
Default Timer (minutes) Auto-off timer used only by Turn On; 0 disables timer arming and timer safety forces off 180

Operating Modes

Mode Description Power State
Off Fireplace completely off OFF
Standby Pilot only, not heating OFF
Manual Fixed flame height - you control flame level directly ON
Smart Thermostat mode - auto-adjusts flame to reach target temperature ON
Eco Energy-saving thermostat mode ON
⚠️ Note: Temperature setpoint control only works in Smart and Eco modes. In Manual mode, the flame level is fixed regardless of temperature.

Control4 Integration

Thermostat Proxy

The driver implements a ThermostatV2 proxy for seamless integration with Control4 interfaces:

  • Heat mode maps to fireplace ON (Manual/Smart/Eco based on default setting)
  • Off mode turns the fireplace completely off
  • Temperature setpoint adjusts the thermostat target (Smart/Eco modes)
  • Fan mode adjusts the blower level
  • The Hold button uses the documented Permanent display value; Hold tap actions set Low, Medium, or High flame where supported
  • Thermostat scheduling is not advertised by this driver

Available Commands

These commands can be used in programming:

Power & Mode

  • Turn On - Turn on using defaults; Default Timer 0 is forced off by timer safety
  • Turn Off - Cancel any active timer and turn fireplace off
  • Set Mode Manual - Send manual mode change only; driver does not adjust flame or timer
  • Set Mode Smart - Send thermostat mode change only; driver does not adjust flame or timer
  • Set Mode Eco - Send eco mode change only; driver does not adjust flame or timer

Flame Control

  • Set Flame Level - Set flame (0-6); switches Smart, Eco, or Off to Manual first
  • Flame Up - Increase flame by 1
  • Flame Down - Decrease flame by 1

Fan Control

  • Set Fan Level - Set fan (0-6)
  • Fan On - Fan to maximum
  • Fan Off - Fan off

Light Control

  • Set Light Level - Set accent light (0-6)
  • Light On - Light to maximum
  • Light Off - Light off

Temperature

  • Set Temperature - Set thermostat target (60-90°F)

Other Controls

  • Toggle Pilot / Pilot On / Pilot Off - Continuous pilot control
  • Toggle Aux - Toggle auxiliary output
  • Toggle Front Flame - Toggle split flow / front flame
  • Set Timer - Set requested auto-off timer minutes; Default Timer is not used
  • Cancel Timer - Cancel active timer; while on, the timer-required safety policy turns the fireplace off

Timer-Required Safety Policy

Manual, Smart, and Eco are on states that require an active auto-off timer. If confirmed device status shows the fireplace on while timer_status is not running, or if timer_status remains unknown after status sync, the driver logs the safety action and sends Turn Off. Timer setup and Turn Off transitions are exempt while timer updates are intentionally suppressed. This changes earlier behavior: Canceling the timer while the fireplace is on, or turning on with Default Timer set to 0, also turns the fireplace off under this policy.

Events

The following events are fired for use in programming:

  • Fireplace Turned On - When confirmed device mode changes from Off/Standby to Manual, Smart, or Eco
  • Fireplace Turned Off - When confirmed device mode changes from Manual, Smart, or Eco to Off/Standby
  • Mode Changed - When confirmed device operating mode changes after initial status baseline
  • Connection Lost - When a previously handshaken connection is lost
  • Connection Restored - When WebSocket handshake completes after being disconnected

Troubleshooting

Connection Issues

  1. Verify the fireplace controller is powered on and connected to WiFi
  2. Confirm the IP address is correct (check your router or use the Proflame app)
  3. Ensure port 88 is not blocked by any firewall
  4. Change the IP Address or Port property, or reload the driver, to force a reconnect
  5. If Strict WebSocket Handshake is On, check Lua output for "Handshake failed" diagnostics
  6. Check WiFi Signal Strength property - poor signal may cause disconnects

Debug Mode

Enable Debug Mode in the driver properties to see detailed logging in Composer Pro's Lua output. Debug levels:

  • Error - Only critical errors
  • Warning - Warnings and errors
  • Info - General information (default)
  • Debug - Detailed debugging info
  • Trace - All messages including raw data

Technical Details

Protocol

  • WebSocket over TCP on port 88
  • No authentication required
  • Keep-alive via PROFLAMEPING/PROFLAMEPONG every 5 seconds
  • Device pushes status updates (no polling required)
  • All outbound control writes use the centralized command-format plan. Non-Turn-Off commands default to the legacy-only format verified on firmware FW: 625.04.673 and can be changed with the Command Format property; Turn Off uses the verified legacy-only plan.

Temperature Encoding

Temperatures are encoded as Fahrenheit × 10. For example:

  • 70°F is sent as "700"
  • 75°F is sent as "750"
  • Range: 60-90°F (sent as 600-900)
✓ Tip: For best results with the thermostat feature, use Smart mode which actively adjusts flame height to maintain the target temperature.
